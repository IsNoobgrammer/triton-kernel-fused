## Requirements

- An NVIDIA GPU with CUDA. Tuned on a **Tesla T4 (`sm_75`)** and verified on a production-grade
  **RTX PRO 6000 Blackwell (`sm_120`)** — see [Hardware and portability](/triton-kernel-fused/concepts/hardware/).
- **Python ≥ 3.10**
- **PyTorch ≥ 2.4** built with CUDA, and **Triton ≥ 3.0** (Triton ships inside the CUDA PyTorch wheel).

The kernels package declares **no hard torch/triton dependency** — you provide a CUDA PyTorch + Triton (most
GPU containers already do), or the setup script below installs one only if it's missing. Nothing ever
reinstalls over a working CUDA build.

## Install

<Tabs>
  <TabItem label="Copy the folder (no install)">
    The kernels need no installation to be *used*. Copy the `kernels/` directory into your project and
    import it directly — it is a self-contained package with no build step.

    ```bash
    cp -r triton-kernel-fused/kernels  your_project/
    ```

    ```python
    from kernels.sm75 import fused_linear_cross_entropy, fused_xsa, FusedMuon, fused_router, moe
    ```
  </TabItem>
  <TabItem label="Bench from a clone">
    Reuse what you have, install only what's missing — so a GPU container that already ships torch is never
    clobbered:

    ```bash
    git clone https://github.com/IsNoobgrammer/triton-kernel-fused
    cd triton-kernel-fused

    python scripts/env_check.py     # is this env ready? (probe only, installs nothing)
    python scripts/setup_env.py     # reuse if ready; else install ONLY what's missing
    python bench.py                 # auto-detects your arch (sm_75 / sm_120 / ...)
    ```

    `setup_env.py` never reinstalls a working torch/triton; on a fresh machine it installs them with a
    selectable CUDA channel (`--cuda cu128`, or `--torch-index <url>`). It uses `uv pip` if `uv` is on PATH,
    else `pip`.
  </TabItem>
  <TabItem label="Existing CUDA environment">
    Already have a CUDA PyTorch + Triton (most GPU containers do)? Nothing to install — run from the repo
    root:

    ```bash
    python bench.py
    ```

    To get an isolated [uv](https://docs.astral.sh/uv/) venv that **reuses** the container's torch instead of
    redownloading it:

    ```bash
    uv venv --system-site-packages   # the venv inherits the container's torch + triton
    uv pip install -e .              # installs only this package; existing torch satisfies the rest
    ```
  </TabItem>
</Tabs>

## Your first kernel

The fused-linear cross-entropy is the easiest win to feel: it computes the same mean cross-entropy as
`F.cross_entropy(hidden @ lm_head.T, labels)` but never materializes the full `(N, V)` logits tensor, so
it trains at vocabulary and sequence sizes where the standard path runs out of memory.

```python
import torch
from kernels.sm75 import fused_linear_cross_entropy

hidden  = torch.randn(16384, 512, device="cuda", dtype=torch.float16, requires_grad=True)
lm_head = torch.randn(81000, 512, device="cuda", dtype=torch.float16, requires_grad=True)
labels  = torch.randint(0, 81000, (16384,), device="cuda")

loss = fused_linear_cross_entropy(hidden, lm_head, labels)   # mean CE, no (N,V) logits materialized
loss.backward()                                              # grad_hidden and grad_lm_head populated

print(loss.item())
```

That is the whole API surface for a kernel: a single function call that returns a tensor with a working
backward. Each kernel page documents its exact signature, tensor shapes, and tuning knobs.

## Verify parity on your machine

Correctness is the gate for every kernel. Before trusting a kernel in a training run, confirm it matches
the eager reference (forward **and** gradients) on your own GPU:

```bash
python bench.py --compile ce_fit     # fused-linear cross-entropy
python bench.py --compile xsa        # XSA
python bench.py --compile muon       # Fused Muon
python bench.py --compile router     # conv MoE router
python bench.py --compile moe        # PolyGLU MoE
```

Each run prints both a speed/memory comparison against the `torch.compile`'d baseline and a gradient
parity check. See [Benchmarking](/triton-kernel-fused/concepts/benchmarking/) for how to read the output.
